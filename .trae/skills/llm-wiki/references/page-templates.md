# 页面模板参考

本文件定义知识库所有页面类型、格式模板及日志格式，供操作时直接引用。

---

## Wiki 页面创建 Prompt 模板

创建任何 wiki 页面时，Agent 必须严格遵循以下规则：

### 1. 文件名

- **人物**：使用通用中文译名，分隔符用 `-`（例：`Boris Cherny.md`、`Jaya Gupta.md`）
- **产品/工具**：无通用中文名则保留英文（例：`Claude-Code.md`、`OpenClaw.md`）
- **概念**：中文描述性短语 4-10 字（例：`AI行业组织护城河.md`、`OpenClaw与Hermes对比.md`）
- 禁止空格，统一使用 `-` 分隔

### 2. 子目录

根据页面类型放入对应目录：
- 人物 → `人物/`
- 产品、工具、项目 → `工具与产品/`
- 概念、想法、框架、对比 → `概念/`
- 来源综合 → `摘要/`

### 3. 摘要特别规则（重要）

**摘要是知识库的素材发现入口。** 下游写作 skill（imitation-writer / script-writer 等）写作前会读取 `index.md`，仅凭摘要判断页面是否与写作主题相关。摘要不合格 = 页面永远不被发现。

规则：
- ≤60 中文字符，独立可读（不依赖标题上下文）
- 必须具体，说清楚「什么人/什么事/什么观点」，避免模糊描述
- **每次更新页面内容后，必须同步检查摘要是否仍然准确**，不准确就更新

好摘要：`Claude Code 之父，Anthropic 工程师，主张「编程已解决」，软件民主化类比印刷术`
差摘要：`关于 AI 编程工具的介绍`

### 4. 必需章节（按顺序）

```markdown
# 页面标题
> 一句话摘要（≤60 中文字符，独立可读，用于 index.md）

## 概述
2-4 段，覆盖 who/what/when/why

## 核心观点 / 关键事实
- 观点或事实 (→ [[来源页面]])
- 每个观点一行，带来源引用

## 相关页面
- [[页面名]] — 一句话关联原因（≤30 字，不重复展开已有内容）

## 争议与数据空白
- 已知批评、局限性、未覆盖内容（概念密集型必填，实体选填）

## 来源
- [来源标题](../sources/batch_xxx/001.md) — YYYY-MM-DD
```

### 5. 交叉引用规则

- 每个页面**至少链接 2 个其他页面**
- 链接处只放一句话关联原因（≤30 字），不重复展开已在其他页面详细描述的内容。例如：`[[AI原生工程团队]] — Fiona 的五大重写框架` 而非复制五大重写的全部内容
- 链接格式：`[[文件名不带扩展名]]`
- 关联原因用 `—` 分隔，不用括号
- 标记矛盾：`> ⚠️ 与 [[来源页面]] 矛盾：具体矛盾描述`

### 6. 禁止行为

- 不要静默覆盖已有信息（应整合并标注变更）
- 不要跳过交叉引用
- 不要使用 HTML 标签（纯 Markdown）
- 不要在来源中使用绝对路径

## 页面类型定义

- **实体页面** — 一个人、产品、公司、地点。分别放入 `人物/` 或 `工具与产品/`
- **概念页面** — 一个想法、框架、理论或对比。放入 `概念/`
- **摘要页面** — 某个来源或主题群的综合。放入 `摘要/`

命名规则：使用中文描述性名称（人物用中文译名，产品无译名则保留英文），分隔符用 `-`。一个主题一个文件。

## 页面格式模板

```markdown
# 页面标题

> 一句话摘要（也用于 index.md）

## 概述

主要内容...

## 关键事实 / 核心观点

- 事实或观点 (→ [[来源页面]])
- 另一个观点

## 相关页面
- [[other-page]] — 关联原因
- [[another-page]] — 关联原因

## 争议与数据空白

- 已知的批评或局限性
- 尚未覆盖或不确定的内容

## 来源
- [来源标题](../sources/folder/file) — YYYY-MM-DD
```

每个页面都必须包含**相关页面**和**来源**。对于概念密集型和主张密集型的页面，添加**争议与数据空白**。

## index.md 格式模板

```markdown
# 知识库目录

_最后更新：YYYY-MM-DD — N 个页面_

## 人物
| 页面 | 摘要 | 更新日期 |
|------|------|----------|
| [[人物名]] | 一句话描述 | YYYY-MM-DD |

## 工具与产品
| 页面 | 摘要 | 更新日期 |
|------|------|----------|
| [[产品名]] | 一句话描述 | YYYY-MM-DD |

## 概念
| 页面 | 摘要 | 更新日期 |
|------|------|----------|
| [[概念名]] | 一句话描述 | YYYY-MM-DD |

## 摘要
| 页面 | 摘要 | 更新日期 |
|------|------|----------|
| [[摘要名]] | 一句话描述 | YYYY-MM-DD |
```

每次查询时首先读取 `index.md`。每次 ingest 后更新它。

## log.md 格式模板

追加写入，永不修改历史条目。

### ingest — 摄入素材

```markdown
## [YYYY-MM-DD HH:MM] ingest | 来源标题
来源：<url 或文件名>
影响页面：topic-a.md（更新）、topic-b.md（新建）、topic-c.md（更新）
---
```

### query — 查询知识库

```markdown
## [YYYY-MM-DD HH:MM] query | 问题摘要
读取页面：index.md、topic-a.md、topic-b.md
回答已保存：是 → analysis-xyz.md / 否
---
```

### lint — 健康检查

```markdown
## [YYYY-MM-DD HH:MM] lint | 健康检查
问题：2 个矛盾、1 个孤立页面、3 个缺失交叉引用
执行操作：...
---
```

### process — 批量处理

```markdown
## [YYYY-MM-DD HH:MM] process | 处理目标
目标：<处理的页面或主题>
读取页面：page-a.md、page-b.md、page-c.md
执行操作：<合并/拆分/重构/去重>
影响页面：page-a.md（重构）、page-b.md（合并至 page-a）、page-c.md（新增交叉引用）
---
```

### digest — 知识消化

```markdown
## [YYYY-MM-DD HH:MM] digest | 消化范围
范围：<时间段或主题范围>
读取页面：index.md、page-a.md、page-b.md、page-c.md
产出页面：digest-xxx.md（新建）
摘要：对 N 个页面进行综合，识别出 X 个核心主题、Y 个待补充缺口
---
```
